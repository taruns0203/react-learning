Common Pitfalls & Fixes

‚ùå PITFALL #1

Controlled without onChange

This creates a read-only input that ignores user keystrokes!

‚ùå Wrong

const [value] = useState("Hello"); <input value={value} /> // User can't type anything!

‚úÖ Fixed

const [value, setValue] = useState("Hello"); <input value={value} onChange={e => setValue(e.target.value)} />
‚ùå PITFALL #2

Undefined Initial State

Starting with undefined then setting a value switches from uncontrolled to controlled!

Initial state: undefined

‚ö†Ô∏è React Warning

A component is changing an uncontrolled input to be controlled. This is likely caused by the value changing from undefined to a defined value.

‚úÖ Fix: Always initialize with empty string

const [value, setValue] = useState(''); // ‚úÖ Not undefined! // OR with API data: <input value={data?.name ?? ''} /> // ‚úÖ Fallback
‚ùå PITFALL #3

defaultValue Doesn't Update

defaultValue only sets the initial value. Changing it later has no effect!

Current defaultValue prop: "Original"

üëÜ Notice: Input stays the same even when defaultValue changes!

‚úÖ Fix: Use key prop to force remount

// When data changes, key forces new component instance <input key={user.id} // ‚úÖ New key = new component defaultValue={user.name} />
